This section contains code for constructing the encoded packets that allow the client to perform certain actions. Like other sections, not every single packet instance is included here. Only a few of the major ones I completed are included, as I felt these were somewhat useful. Before quitting I would've completed the movement/"C" packet parser, but I admittedly never did that before quitting. Otherwise pretty much everything that is useful is here, and even then the "C" packet is absolutely not all that useful or necessary. Note that you'll see a lot of numbers like 192 and 254 appear, this how to do with how the arras protocol encodes and handles strings. Due to the logic that is applied, depending on if a string length is < 32, there are different behaviors. For < 192, the length will be recorded as 192 + length, which is why that is a case in a few of these. Otherwise, the sort of length indication byte is set to 254 and some length data is handled afterward. So yeah that's why. I forget the exact details since I haven't really looked at this in a while but I'm sure you can figure it out.
